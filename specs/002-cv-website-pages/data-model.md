# Data Model: Static CV Website Pages

**Feature**: 002-cv-website-pages | **Date**: 2026-04-13

## Overview

The site uses a single JSON artifact (`public/content/site-content.json`) as the data layer. This file is generated at build time from source text files and is consumed by Next.js pages via static import.

## Entities

### SiteContent (root)

The top-level JSON object. Generated by `tools/build-content.ts`.

| Field         | Type                                            | Source                                          | Description                                  |
| ------------- | ----------------------------------------------- | ----------------------------------------------- | -------------------------------------------- |
| schemaVersion | `"1"` (const)                                   | Build tool                                      | Schema version for forward compatibility     |
| generatedAt   | ISO 8601 datetime                               | Build tool                                      | Build timestamp                              |
| person        | Person                                          | `specs-input/cv/summary/meta.txt`               | Site owner identity                          |
| about         | `{ text: LocalizedText }`                       | `specs-input/about/<lang>.txt`                  | About page content                           |
| notFound      | `{ title: LocalizedText, text: LocalizedText }` | `specs-input/not-found/<lang>.txt` + build tool | 404 page content                             |
| technologies  | `Record<string, TechItem[]>`                    | CV entries × `tech-categories.json`             | Tech by category with years (non-empty only) |
| cv            | CVData                                          | `specs-input/cv/`                               | All CV sections                              |
| ui            | UIStrings                                       | Build tool (hardcoded)                          | All translatable UI labels                   |

### Person

| Field | Type   | Source                                      |
| ----- | ------ | ------------------------------------------- |
| name  | string | `specs-input/cv/summary/meta.txt` → `Name:` |
| role  | string | `specs-input/cv/summary/meta.txt` → `Role:` |

### LocalizedText

| Field | Type   |
| ----- | ------ |
| en    | string |
| sv    | string |

### TechItem

| Field | Type    | Description                                                                                    |
| ----- | ------- | ---------------------------------------------------------------------------------------------- |
| name  | string  | Canonical technology name (from `tech-categories.json`, or raw CV name if uncategorized)       |
| years | integer | Total years of experience, computed by summing CV entry durations that include this technology |

### CVData

| Field       | Type          | Description                                                                             |
| ----------- | ------------- | --------------------------------------------------------------------------------------- |
| summary     | LocalizedText | CV summary text                                                                         |
| education   | CVEntry[]     | Entries where `type === "education"`, sorted newest-first                               |
| roles       | CVEntry[]     | Entries where `type ∈ {employee, consultant, voluntary, national}`, sorted newest-first |
| assignments | CVEntry[]     | Entries where `type === "assignment"`, sorted newest-first                              |

### CVEntry

| Field        | Type          | Nullable | Description                                                                  |
| ------------ | ------------- | -------- | ---------------------------------------------------------------------------- |
| id           | string        | No       | Directory name (e.g., `200002_Axis`)                                         |
| type         | enum          | No       | `education \| employee \| consultant \| voluntary \| national \| assignment` |
| period       | Period        | No       | Time range                                                                   |
| sortKey      | string        | No       | `YYYYMM` for reverse-chrono sort                                             |
| organization | string        | No       | Employer or institution name                                                 |
| client       | string        | Yes      | Client name (assignments only, typically)                                    |
| role         | string        | No       | Job title / role name                                                        |
| location     | string        | No       | City                                                                         |
| locationType | enum          | No       | `onsite \| hybrid \| remote`                                                 |
| title        | LocalizedText | No       | Display label for education/assignment items                                 |
| description  | LocalizedText | No       | Full description text                                                        |
| technologies | string[]      | No       | Technology names from meta.txt `Technologies` field; empty array if none     |

### Period

| Field | Type   | Nullable | Description                          |
| ----- | ------ | -------- | ------------------------------------ |
| raw   | string | No       | Original period string from meta.txt |
| start | string | No       | `YYYY` or `YYYYMM`                   |
| end   | string | Yes      | `YYYY` or `YYYYMM`, null if ongoing  |

### ContactMessage (runtime only, not persisted)

| Field   | Type   | Required | Description     |
| ------- | ------ | -------- | --------------- |
| name    | string | Yes      | Sender name     |
| email   | string | Yes      | Sender email    |
| subject | string | Yes      | Message subject |
| message | string | Yes      | Message body    |

Sent via EmailJS — not stored in site-content.json.

## Relationships

```
specs-input/cv/          ──build──▶  site-content.json
specs-input/about/       ──build──▶  site-content.json
specs-input/not-found/   ──build──▶  site-content.json
tech-categories.json     ──build──▶  site-content.json (technologies)

site-content.json        ──import──▶ loadContent() ──▶ All pages

CVEntry.type             ──partitions──▶ education | roles | assignments
tech-categories.json × CVEntry.technologies ──intersect + sum durations──▶ technologies (+ "Other" last)
```

## Validation

- **Build time**: `tools/validate-content.ts` validates `site-content.json` against `contracts/site-content.schema.json` (JSON Schema 2020-12).
- **Build time**: `tools/build-content.ts` validates source file structure (required fields, period format, type enum).
- **Runtime**: No validation — content is trusted as build artifact.

## State Transitions

### ContactMessage lifecycle

```
idle → sending → success
                → error (retry returns to idle)
```

### CV list item

```
collapsed (default) ←→ expanded (on click toggle)
```

### CV section show-all

```
collapsed (show 5) → expanded (show all)
```

Note: show-all state resets when search is active (all matching items shown).
